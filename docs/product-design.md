## 1. 产品概述

### 1.1 产品定位

Tydora 是一款基于 Tauri v2 + React 19 的桌面 Markdown 编辑器，专为知识管理和笔记场景设计。产品融合了 WYSIWYG 与源码两种编辑模式，提供 Obsidian 风格的双向链接系统，并支持将笔记发布为静态网站。

### 1.2 核心理念

- **双模式编辑**：即时渲染（IR）模式提供所见即所得的编辑体验，源码（SV）模式满足精确控制需求
- **仓库级知识管理**：以"仓库（Vault）"为单位组织文件，支持多仓库切换
- **双向链接**：`[[wiki-link]]` 语法构建知识网络，自动生成反向链接关系
- **发布即用**：一键将笔记仓库发布为可部署的静态网站

### 1.3 目标平台

| 平台 | 优先级 | 打包格式 |
|------|--------|----------|
| Windows | 主要 | NSIS 安装包 |
| macOS | 支持 | DMG |
| Linux | 支持 | AppImage |

### 1.4 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 19 + TypeScript |
| 构建工具 | Vite 6 |
| 编辑器引擎 | TipTap 3.x（WYSIWYG）+ CodeMirror 6（源码） |
| Markdown 序列化 | tiptap-markdown |
| 桌面框架 | Tauri v2（Rust 后端） |
| 可视化 | D3.js（图谱）、markmap（思维导图）、mermaid（图表） |
| 语法高亮 | highlight.js、lowlight |

## 3. 编辑器系统

### 3.1 双模式架构

Tydora 提供两种编辑模式，通过 `Ctrl+/` 快捷键切换：

| 模式 | 标识 | 引擎 | 适用场景 |
|------|------|------|----------|
| 即时渲染 (IR) | `"ir"` | TipTap 3.x | 日常写作、WYSIWYG 体验 |
| 源码模式 (SV) | `"sv"` | CodeMirror 6 | 精确控制 Markdown 语法 |

默认模式可在设置中配置（编辑器 → 默认模式）。

### 3.2 TipTap 编辑器

基于 TipTap 3.x 构建，注册 22+ 扩展，通过 `forwardRef` + `useImperativeHandle` 暴露编辑器 API。

#### 核心扩展

| 扩展 | 用途 |
|------|------|
| StarterKit | 核心扩展集合（段落、代码块、链接、格式等） |
| Bold / Italic / Strike / Code | 行内格式化 |
| Blockquote | 引用块（扩展支持 Callout 序列化） |
| BulletList / OrderedList / ListItem | 列表 |
| Heading | H1-H6 标题 |
| Placeholder | 占位符文本 |
| CodeBlockLowlight | 语法高亮代码块 |
| Image | 图片（支持 inline、base64、本地文件） |
| Link | 超链接 |
| Table / TableRow / TableCell / TableHeader | 可调整大小的表格 |
| TaskList / TaskItem | 任务列表（支持嵌套） |
| Highlight | 文本高亮 |
| Typography | 智能排版 |
| Markdown (tiptap-markdown) | Markdown ↔ HTML 双向转换 |

#### 自定义扩展

| 扩展 | 文件 | 功能 |
|------|------|------|
| WikiLink | `extensions/wiki-link.ts` | `[[双向链接]]` 节点，支持 `[[note#heading\|alias]]` 语法 |
| SearchHighlight | `extensions/search-highlight.ts` | 搜索结果高亮装饰器 |
| CodeBlockToolbar | `extensions/code-block-toolbar.ts` | 代码块浮动工具栏（语言选择、复制、折叠、行号） |
| TableFloatingToolbar | `extensions/table-floating-toolbar.ts` | 表格浮动工具栏（对齐、行列操作） |
| Frontmatter | `extensions/frontmatter.ts` | YAML frontmatter 块，语法高亮，可折叠 |
| Callout | `extensions/callout.ts` | GitHub 风格提示块（15 种类型） |
| Mermaid | `extensions/mermaid.ts` | Mermaid 图表实时预览，内嵌 CodeMirror 编辑器 |

#### Callout 类型

支持 15 种 GitHub 风格的 Callout，每种都有独特图标和标签：

| 类型 | 标签 | 语法 |
|------|------|------|
| note | 注意 | `[!NOTE]` |
| tip | 提示 | `[!TIP]` |
| important | 重要 | `[!IMPORTANT]` |
| warning | 警告 | `[!WARNING]` |
| caution | 注意 | `[!CAUTION]` |
| abstract | 摘要 | `[!ABSTRACT]` |
| info | 信息 | `[!INFO]` |
| success | 成功 | `[!SUCCESS]` |
| question | 问题 | `[!QUESTION]` |
| failure | 失败 | `[!FAILURE]` |
| danger | 危险 | `[!DANGER]` |
| bug | Bug | `[!BUG]` |
| example | 示例 | `[!EXAMPLE]` |
| quote | 引用 | `[!QUOTE]` |
| FAQ | FAQ | `[!FAQ]` |

Callout 支持 `+`/`-` 修饰符控制折叠状态。

#### 编辑器 API（EditorHandle）

通过 `useImperativeHandle` 暴露的方法：

| 方法 | 说明 |
|------|------|
| `getValue()` | 获取 Markdown 内容 |
| `setValue(val)` | 设置内容（带内部标志防止循环） |
| `insertTextAtCursor(text)` | 在光标处插入文本 |
| `replaceRangeWithWikiLink(from, note, heading?, display?)` | 替换范围为 WikiLink 节点 |
| `resize()` | 重新聚焦编辑器 |
| `highlightSearch(query)` | 搜索高亮 |
| `clearHighlight()` | 清除高亮 |
| `executeCommand(name)` | 执行命名命令 |
| `scrollToHeading(text, line)` | 滚动到标题 |
| `scrollToLine(line)` | 滚动到行 |
| `getCursorOffset()` | 获取光标位置 |
| `isSourceMode()` | 检查当前模式 |

### 3.3 CodeMirror 编辑器

源码模式使用 CodeMirror 6，功能包括：

- **语言自动检测**：根据文件扩展名识别 20+ 语言（md、js、ts、tsx、jsx、html、css、json、xml、yaml 等）
- **语法高亮**：两套主题 — `markdownHighlighting`（标题大小、强调等）和 `codeHighlighting`（程序代码）
- **编辑功能**：行号、活动行高亮、代码折叠、括号匹配、自动关闭括号、自动补全、搜索高亮、自动换行
- **主题同步**：监听 `code-theme-changed` 事件动态切换高亮主题

### 3.4 Markdown 序列化

使用 `tiptap-markdown` 扩展实现双向转换：

```typescript
// 获取 Markdown
const md = (editor.storage as any).markdown.getMarkdown();

// 设置内容（HTML 或 Markdown）
editor.commands.setContent(htmlOrMarkdown);
```

### 3.5 图片处理

- **粘贴拦截**：拦截 `image/*` 剪贴板内容，自动保存到 vault 并插入图片节点
- **拖拽支持**：拖拽图片文件到编辑器自动保存并插入
- **图片存储**：通过 ImageManager 保存到 vault-assets 目录
- **路径解析**：自定义 Image NodeView 解析相对路径，支持本地文件和远程图片
- **远程图片**：通过 Rust `fetch_remote_image` 代理加载外部图片

### 3.6 链接编辑行为（IR 模式）

- **点击链接**：将渲染的链接转换为 Markdown 源码 `[text](url)` 进行编辑
- **点击外部**：恢复渲染的链接
- **Ctrl+Click**：打开链接（外部链接通过 `open_url`，本地文件通过 `open_file`）
- **Escape**：恢复正在编辑的链接

### 3.7 表格浮动工具栏

选中表格时显示浮动工具栏，功能包括：

| 功能 | 说明 |

| 网格选择器 | 8×10 网格调整表格尺寸 |

### 3.8 代码块工具栏

代码块上方显示浮动工具栏：

| 功能 | 说明 |
|------|------|

编辑器内右键显示图标网格布局菜单：

## 5. WikiLink 知识管理系统

### 5.1 语法规范

Tydora 支持 Obsidian 风格的 WikiLink 语法：

| 语法 | 说明 | 示例 |
|------|------|------|
| `[[note]]` | 链接到笔记 | `[[我的笔记]]` |
| `[[note#heading]]` | 链接到笔记的特定标题 | `[[我的笔记#第一章]]` |
| `[[note\|alias]]` | 带别名的链接 | `[[我的笔记\|别名]]` |
| `[[note#heading\|alias]]` | 带标题和别名 | `[[我的笔记#第一章\|第一章]]` |
| `![[embed]]` | 嵌入笔记内容 | `![[我的笔记]]` |
| `[[[note]]]` | 中括号语法（中文输入法兼容） | `[[[我的笔记]]]` |

### 5.2 自动补全

输入 `[[` 时触发自动补全：

| 特性 | 说明 |
|------|------|
| 触发条件 | 输入 `[[`（中英文括号均支持） |
| 搜索方式 | 模糊匹配笔记名称 |
| 键盘导航 | ↑/↓ 或 Ctrl+J/K 选择 |
| 确认选择 | Enter |
| 关闭 | Escape |
| 无匹配 | 显示"无匹配，Enter 创建新笔记" |
| 定位 | 视口感知定位（上方/下方自动切换） |

### 5.3 链接索引服务

`LinkIndexService` 单例维护三个核心索引：

| 索引 | 类型 | 说明 |
|------|------|------|
| `outlinks` | `Map<filePath, noteName[]>` | 每个文件链接到哪些笔记 |
| `backlinks` | `Map<noteName, filePath[]>` | 每个笔记被哪些文件链接 |
| `fileByName` | `Map<noteName, filePath>` | 按笔记名快速查找文件 |

操作方法：
- `buildIndex(vaultPath)` — 全量重建（批量读取所有 .md 文件）
- `updateFileLinks(filePath, vaultPath)` — 增量更新单个文件
- `removeFile(filePath)` — 清理已删除文件的索引
- `getBacklinks(noteName)` / `getOutlinks(filePath)` — 查询操作
- `findFileByNoteName(noteName)` — 精确匹配，然后大小写不敏感匹配（最短路径优先）
- `searchNotes(query)` — 模糊搜索用于自动补全
- `serialize()` / `deserialize(json)` — JSON 序列化用于跨窗口传输

### 5.4 反向链接面板

可折叠面板显示当前文件的链接关系：

| 区域 | 说明 |
|------|------|
| 反向链接 | 哪些文件通过 `[[note]]` 引用了当前笔记，附带上下文段落（最多 200 字符） |
| 出链 | 当前文件引用了哪些笔记 |

点击链接跳转到对应文件。

### 5.5 链接存在性标记

- 有效链接：`data-exists="true"`，渲染为可点击的样式化链接
- 无效链接：`data-exists="false"`，渲染为不同颜色（表示目标笔记不存在）

### 5.6 嵌入支持

`![[embed]]` 语法嵌入笔记内容：
- 异步加载目标笔记内容
- 显示前 500 字符
- 渲染为引用块样式

### 5.7 文件监听与索引同步

基于 Rust `notify` crate 的实时文件监听：
- Vault 切换时自动启动监听
- 文件变更事件防抖处理（300ms）
- 自动增量更新 LinkIndexService 索引
- 组件卸载或 Vault 切换时停止监听

## 6. 多窗口架构

### 6.1 窗口类型

| 窗口 | 触发方式 | 默认大小 | 特性 |
|------|----------|----------|------|
| 主窗口 | 应用启动 | 1200×800 | 编辑器、侧栏等核心 UI |
| 设置窗口 | `open_settings_window` | 800×600 | 独立设置面板，无装饰 |
| 文件窗口 | `open_file_in_new_window` | 1200×800 | 独立编辑文件 |
| 思维导图窗口 | `open_mindmap_window` | 900×600 | Markmap 可视化 |
| 知识图谱窗口 | `open_graph_window` | 1000×700 | D3.js 力导向图 |

### 6.2 窗口特性

- **无装饰窗口**：所有窗口使用自定义标题栏（最小化、最大化/还原、关闭）
- **窗口状态持久化**：位置、大小、最大化状态保存到 localStorage
- **多显示器支持**：启动时验证窗口位置是否在可见显示器上
- **防抖保存**：窗口移动和调整大小时 300ms 防抖保存状态
- **单实例控制**：设置、思维导图、图谱窗口已存在时聚焦而非新建

### 6.3 跨窗口通信

- **Tauri 事件**：`theme-changed`（主题同步）、`mindmap-content-update`（思维导图内容同步）、`open-file`（文件窗口打开文件）
- **localStorage**：`zmd-mindmap-content`（思维导图内容缓存）
- **JSON 序列化**：LinkIndexService 的 `serialize()`/`deserialize()` 跨窗口传输链接索引
