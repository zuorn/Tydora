# 双链（双向链接）功能需求文档

 **文档版本** ：v1.0
 **编制日期** ：2026年6月25日
 **文档状态** ：草稿

## 一、项目概述

### 1.1 项目背景

在知识管理场景中，传统笔记工具采用层级化存储，难以建立知识间的非线性关联[](https://developer.baidu.com/article/detail.html?id=3562473)。Obsidian 通过双向链接机制颠覆了线性笔记结构——当用户在笔记 A 中插入 `[[笔记B]]` 时，系统会自动在笔记 B 中生成指向笔记 A 的反向链接，从而构建起互联的知识网络[](https://developer.baidu.com/article/detail.html?id=3562473)。

本项目旨在为自有编辑器/笔记产品实现同等的双链功能，使用户能够在笔记之间建立双向关联，并通过反向链接面板和知识图谱实现知识的可视化探索。

### 1.2 项目目标

* 支持 `[[笔记名]]` 语法的解析、渲染和交互
* 自动维护每个笔记的 **出链** （Outgoing Links）和 **入链/反向链接** （Backlinks）[](https://wenku.csdn.net/doc/7n7yqo7m0n)
* 提供反向链接面板，展示引用当前笔记的所有其他笔记
* 提供知识图谱视图，可视化展示笔记间的链接网络[](https://developer.baidu.com/article/detail.html?id=3562473)
* 支持链接到不存在的笔记（占位链接），点击后自动创建

### 1.3 适用范围

本需求文档适用于：

* Web 端 Markdown 编辑器
* 桌面端笔记应用
* 任何需要实现双向链接功能的文档/知识管理产品

## 二、功能需求

### 2.1 核心功能

#### FR-01：内部链接创建

| 项目               | 说明                                                                              |
| ------------------ | --------------------------------------------------------------------------------- |
| **功能描述** | 用户可通过 `[[笔记名]]` 语法在当前笔记中创建指向另一篇笔记的内部链接            |
| **输入方式** | 手动输入 `[[` 触发自动补全弹出框；或选中文本后按特定快捷键（如 `[[`）快速创建 |
| **自动补全** | 输入 `[[` 后，系统弹出候选笔记列表，支持模糊匹配搜索                            |
| **验收标准** | 输入 `[[` 后 200ms 内弹出候选列表；候选列表包含所有可链接的笔记                 |

#### FR-02：链接跳转

| 项目               | 说明                                                       |
| ------------------ | ---------------------------------------------------------- |
| **功能描述** | 点击渲染后的内部链接，跳转到目标笔记                       |
| **交互方式** | 点击链接打开目标笔记（同一窗口或新标签页，取决于产品设计） |
| **验收标准** | 点击链接后 500ms 内完成目标笔记的加载和渲染                |

#### FR-03：反向链接（Backlinks）自动维护

| 项目               | 说明                                                                                                                                                                |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **功能描述** | 当笔记 A 创建指向笔记 B 的链接时，系统自动在笔记 B 的反向链接列表中记录笔记 A[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks) |
| **更新时机** | 笔记保存时触发索引更新；支持实时或准实时更新                                                                                                                        |
| **数据结构** | 每个笔记维护一个入链列表（来源笔记列表），包含所有引用它的笔记[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks)                |
| **验收标准** | 笔记 A 链接到笔记 B 并保存后，笔记 B 的反向链接面板中立即（或 1 秒内）出现笔记 A                                                                                    |

#### FR-04：反向链接面板

| 项目               | 说明                                                           |
| ------------------ | -------------------------------------------------------------- |
| **功能描述** | 在笔记详情页侧边栏或底部展示该笔记的反向链接列表               |
| **展示内容** | 引用当前笔记的所有笔记名称、来源上下文（引用所在段落预览）     |
| **交互方式** | 点击反向链接列表中的条目，跳转到对应的源笔记                   |
| **验收标准** | 反向链接面板正确显示所有引用当前笔记的笔记列表，无遗漏、无重复 |

#### FR-05：链接到不存在的笔记（占位链接）

| 项目               | 说明                                                                   |
| ------------------ | ---------------------------------------------------------------------- |
| **功能描述** | 允许用户链接到一个尚未创建的笔记，链接以特殊样式（如下划线或灰色）显示 |
| **自动创建** | 点击占位链接时，自动创建对应的空白笔记                                 |
| **验收标准** | 点击占位链接后，系统在 1 秒内创建新笔记并打开                          |

#### FR-06：链接到标题（锚点链接）

| 项目               | 说明                                                            |
| ------------------ | --------------------------------------------------------------- |
| **功能描述** | 支持 `[[笔记名#标题名]]` 语法，跳转到目标笔记中的指定标题位置 |
| **解析规则** | 解析 `#` 后的内容作为目标笔记内的标题锚点                     |
| **验收标准** | 点击锚点链接后，正确定位到目标笔记中对应的标题位置              |

#### FR-07：自定义显示文本（别名）

| 项目               | 说明                                                                  |
| ------------------ | --------------------------------------------------------------------- |
| **功能描述** | 支持 `[[笔记名\|自定义文本]]` 语法，链接显示为自定义文本而非笔记原名 |
| **验收标准** | 渲染后链接显示为自定义文本，点击后仍正确跳转到目标笔记                |

#### FR-08：嵌入内容（Transclusion）

| 项目               | 说明                                                                                                       |
| ------------------ | ---------------------------------------------------------------------------------------------------------- |
| **功能描述** | 支持 `![[笔记名]]` 语法，将另一篇笔记的完整内容嵌入到当前笔记中[](https://wenku.csdn.net/doc/7n7yqo7m0n) |
| **同步更新** | 源笔记内容更新时，嵌入的内容自动同步                                                                       |
| **验收标准** | 源笔记内容修改并保存后，嵌入位置的内容在 2 秒内自动更新                                                    |

## 三、技术方案建议

### 3.1 整体架构

双链功能的核心是**反向索引（Backlink Indexing）机制**[](https://wenku.csdn.net/doc/7n7yqo7m0n)——系统在后台持续扫描所有笔记中的 `[[]]` 语法标记，动态构建有向图，并自动维护每个节点的入链与出链元数据[](https://wenku.csdn.net/doc/7n7yqo7m0n)。

**text**

```
┌─────────────────────────────────────────────────────────────┐
│                      编辑器层                               │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │ 语法解析器   │  │ 自动补全    │  │ 链接渲染器          │ │
│  │ (Remark插件) │  │ (搜索服务)  │  │ (React/Vue组件)     │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                      索引层                                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │ wikilinks   │  │ backlinks   │  │ 图数据库            │ │
│  │ _index      │  │ _index      │  │ (Graph/NetworkX)    │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                      存储层                                 │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  笔记文件 (Markdown) + 索引缓存 (JSON/SQLite)          │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 数据结构设计

#### 链接索引（Link Index）

参考 Obsidian 的数据结构设计[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks)：

**typescript**

```
interface LinkIndex {
  // 出链索引：每个笔记 → 它包含的所有 wikilink 列表
  wikilinks_index: Record<string, string[]>;      // key: 笔记文件名, value: 目标笔记名列表（含重复）

  // 去重出链索引
  unique_wikilinks_index: Record<string, string[]>; // key: 笔记文件名, value: 去重后的目标笔记名列表

  // 反向链接索引：每个笔记 → 所有引用它的笔记列表
  backlinks_index: Record<string, string[]>;       // key: 笔记名, value: 来源笔记名列表（含重复）
}
```

**关键特性**[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks)：

* 出链索引以**文件名**为键（仅已存在的笔记才能包含出链）
* 反向链接索引以**笔记名**为键（包含被引用但尚未创建的不存在笔记）
* 列表默认保留重复项（若笔记 A 三次链接到笔记 B，笔记 B 的反向链接列表中包含笔记 A 三次）
* 空列表（`[]`）表示该方向无链接

#### 笔记元数据（Note Metadata）

**typescript**

```
interface NoteMetadata {
  id: string;                    // 笔记唯一标识
  filename: string;              // 文件名（含扩展名）
  path: string;                  // 文件路径
  title: string;                 // 笔记标题
  aliases?: string[];            // 别名列表（Frontmatter 中定义）
  outlinks: string[];            // 出链列表（去重）
  backlinks: string[];           // 入链列表（去重）
  tags: string[];                // 标签
  created_at: Date;
  updated_at: Date;
}
```

### 3.3 核心模块设计

#### 3.3.1 语法解析模块

Obsidian 的 `[[]]` 双括号链接并非标准 Markdown 语法[](https://juejin.cn/post/7286266688192954368?from=search-suggest)。实现时需在 Markdown 解析流程中插入自定义插件。

**解析流程**[](https://juejin.cn/post/7286266688192954368?from=search-suggest)：

**text**

```
Markdown 文本
    ↓
[Remark 解析] → Markdown AST (mdast)
    ↓
[Remark 插件：双链转换] → 将 Text 节点中的 [[xxx]] 转换为 Link 节点
    ↓
[Remark 转 Rehype] → HTML AST (hast)
    ↓
[Rehype 插件] → 安全过滤、样式注入
    ↓
HTML 字符串 → 渲染
```

 **核心解析逻辑** （基于 Remark 插件）[](https://juejin.cn/post/7286266688192954368?from=search-suggest)：

**javascript**

```
const parenthesisRegexExclusive = /(?<=\[\[).*?(?=\]\])/g;
// 匹配 [[xxx]] 中的 xxx 内容
```

在 AST 遍历中，识别 `Text` 类型节点，用正则匹配 `[[]]` 模式，将其替换为 `Link` 类型节点[](https://juejin.cn/post/7286266688192954368?from=search-suggest)。

#### 3.3.2 索引服务模块

**typescript**

```
interface IndexService {
  // 全量索引构建
  buildFullIndex(): Promise<void>;

  // 增量更新（单篇笔记变更时触发）
  updateIndex(noteId: string): Promise<void>;

  // 获取笔记的出链
  getOutlinks(noteId: string): string[];

  // 获取笔记的反向链接
  getBacklinks(noteId: string): string[];

  // 搜索笔记（用于自动补全）
  searchNotes(query: string): NoteMetadata[];

  // 检测失效链接
  detectBrokenLinks(): BrokenLinkReport;
}
```

**typescript**

interface AutocompleteService {

  trigger(query: string, context: EditorContext): Suggestion[];

| 模块          | 推荐方案                                                                           | 备选方案                |
| ------------- | ---------------------------------------------------------------------------------- | ----------------------- |

| 索引存储      | SQLite / IndexedDB                                                                 | JSON 文件               |

### 3.5 关键实现要点

1. **双括号文本转换**[](https://juejin.cn/post/7286266688192954368?from=search-suggest)：通过 Remark 插件将 `[[]]` 语法转换为标准 Markdown 链接或自定义 HTML 元素。
2. **反向链接的自动维护**[](https://wenku.csdn.net/doc/7n7yqo7m0n)：每次笔记保存时，重新扫描该笔记中的所有 `[[]]` 语法，更新出链索引；同时遍历所有笔记，重建指向该笔记的反向链接索引。
3. **性能优化** ：当笔记数量超过 10,000 个时，自动补全功能应切换至更高效的搜索算法。
4. **链接校验** ：定期（如每次启动时）扫描所有链接，标记目标笔记已被删除或重命名的失效链接[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks)。

## 四、用户界面需求

### 4.1 编辑器内链接渲染

* 内部链接以特定样式（如蓝色文字 + 下划线）渲染，与普通文本区分
* 悬停时显示目标笔记的预览（Tooltip）
* 占位链接（目标笔记不存在）以灰色/虚线样式显示

### 4.2 反向链接面板

* 位置：笔记详情页右侧边栏或底部
* 分组：按来源笔记分组，显示引用上下文的预览片段
* 计数：显示当前笔记被引用的总次数

### 4.3 知识图谱视图

* 节点：每个笔记为一个节点
* 边：每个链接关系为一条边
* 交互：拖拽画布、滚轮缩放、点击节点跳转
* 筛选：按标签、路径、链接深度等条件筛选

## 六、风险评估

| 风险                     | 影响 | 缓解措施                                    |
| ------------------------ | ---- | ------------------------------------------- |
| 大规模笔记仓库性能下降   | 高   | 采用增量索引、分页加载、Web Worker 异步处理 |
| 循环链接导致图谱渲染卡顿 | 中   | 限制图谱渲染深度、检测并标记循环引用        |
| 笔记重命名导致链接失效   | 中   | 实现链接自动更新机制 + 失效链接检测报告     |
| 与其他 Markdown 语法冲突 | 低   | 严格限定 `[[]]` 的解析范围，避免误匹配    |

| **双链/双向链接**             | 在两个笔记之间建立的互相引用的链接关系                                                                                 |
| **Wikilink**                  | 使用 `[[]]` 语法创建的内部链接[](https://deepwiki.com/mfarragher/obsidiantools/4.4-working-with-links-and-backlinks) |
| **出链（Outgoing Link）**     | 从当前笔记指向其他笔记的链接[](https://wenku.csdn.net/doc/7n7yqo7m0n)                                                  |
| **反向链接/入链（Backlink）** | 从其他笔记指向当前笔记的链接[](https://wenku.csdn.net/doc/7n7yqo7m0n)                                                  |
| **知识图谱（Graph View）**    | 以图结构可视化展示笔记间链接关系的视图[](https://developer.baidu.com/article/detail.html?id=3562473)                   |
| **占位链接**                  | 指向不存在笔记的链接，以特殊样式显示                                                                                   |
