# 编辑器设置方案

> 基于 [Vditor 开发文档](https://ld246.com/article/1549638745630) 及 Tydora 项目现状，设计在「设置」面板中新增「编辑器」标签页的完整方案。

---

## 一、现状分析

### 1.1 当前设置面板结构

`Settings.tsx` 已有 6 个标签页：

| 标签 | 功能 |
|------|------|
| 通用 | 外观模式、字体、字号、自动保存 |
| 主题 | 应用主题切换 |
| 快捷键 | 快捷键自定义 |
| 思维导图 | 思维导图布局/动画/颜色参数 |
| 图像 | 图片存储模式与命名格式 |
| 关于 | 版本信息 |

### 1.2 当前 Vditor 硬编码配置

`VditorEditor.tsx` 初始化时大量参数为**硬编码**，用户无法调整：

```typescript
// 硬编码项一览
mode: mode,              // 由 App.tsx 的 viewMode 传入（ir/wysiwyg/sv）
typewriterMode,          // 由 App.tsx 传入，未持久化
cache: { enable: false },           // 缓存关闭
counter: { enable: false },         // 字数统计关闭
resize: { enable: false },          // 大小拖拽关闭
outline: { enable: false },         // 大纲关闭
preview: {
  maxWidth: 800,                    // 预览最大宽度
  markdown: {
    codeBlockPreview: true,         // 代码块渲染
    mathBlockPreview: true,         // 数学公式渲染
    footnotes: true,                // 脚注
    gfmAutoLink: true,              // 自动链接
    // autoSpace, fixTermTypo, toc, sanitize, mark, sup, sub 等均未暴露
  },
  hljs: {
    style: "atom-one-light/dark",   // 代码主题跟随应用主题
    enable: true,
    // lineNumber 未暴露
  },
},
```

### 1.3 现存问题

1. **编辑模式不持久化**：`viewMode` 和 `typewriterMode` 存在 App.tsx 的 React state 中，刷新后重置为默认值
2. **Markdown 解析选项不可调**：用户无法控制自动空格、脚注、XSS 过滤等
3. **代码高亮选项不可调**：代码行号、代码主题风格无法自定义
4. **编辑器行为选项不可调**：缓存、字数统计、大纲、打字机模式等无法切换
5. **通用设置中字体/字号与编辑器设置混在一起**：`GeneralSettings` 中包含了本应属于编辑器的 `fontSize`/`editorFont`

## 二、设计方案

### 2.1 新增「编辑器」标签页

在 `Settings.tsx` 的 `tabs` 数组中，在「快捷键」之后插入「编辑器」标签：

```
通用 → 主题 → 快捷键 → 编辑器（新增）→ 思维导图 → 图像 → 关于
```

### 2.2 编辑器设置数据结构

定义 `EditorSettings` 接口，涵盖 Vditor 可配置选项中**对桌面用户有意义**的部分：

```typescript
// src/Settings.tsx

interface EditorSettings {
  // ── 编辑模式 ──
  /** 默认编辑模式 */
  defaultMode: "wysiwyg" | "ir" | "sv";
  /** 打字机模式 */
  typewriterMode: boolean;

  // ── 编辑行为 ──
  /** 启用缓存（localStorage） */
  cache: boolean;
  /** 启用字数统计 */
  counter: boolean;
  /** 字数统计类型 */
  counterType: "markdown" | "text";
  /** 字数上限（0 = 不限制） */
  counterMax: number;
  /** 启用大小拖拽 */
  resize: boolean;
  /** 启用大纲 */
  outline: boolean;
  /** 大纲位置 */
  outlinePosition: "left" | "right";

  // ── Markdown 渲染 ──
  /** 自动空格（中西文之间） */
  autoSpace: boolean;
  /** 自动链接 */
  gfmAutoLink: boolean;
  /** 自动矫正术语 */
  fixTermTypo: boolean;
  /** 脚注 */
  footnotes: boolean;
  /** 插入目录 */
  toc: boolean;
  /** 段落开头空两格 */
  paragraphBeginningSpace: boolean;
  /** XSS 过滤 */
  sanitize: boolean;
  /** 代码块渲染预览（wysiwyg/ir） */
  codeBlockPreview: boolean;
  /** 数学公式块渲染预览（wysiwyg/ir） */
  mathBlockPreview: boolean;
  /** mark 标记语法 */
  mark: boolean;
  /** 上标语法 */
  sup: boolean;
  /** 下标语法 */
  sub: boolean;

  // ── 代码高亮 ──
  /** 代码行号 */
  codeLineNumber: boolean;
  /** 代码主题（可独立于应用主题） */
  codeTheme: "auto" | "github" | "atom-one-light" | "atom-one-dark" | "vs" | "nord" | "monokai" | "dracula" | "solarized-light" | "solarized-dark";

  // ── 预览 ──
  /** 预览区域最大宽度 */
  previewMaxWidth: number;  // 600–1200

  // ── 数学公式 ──
  /** 数学公式引擎 */
  mathEngine: "KaTeX" | "MathJax";

  // ── 链接行为 ──
  /** 链接点击是否在新窗口打开 */
  linkOpenNewTab: boolean;
}
```

### 2.4 持久化

使用与其他设置一致的 localStorage 方案：

```typescript
const EDITOR_SETTINGS_KEY = "zmd-editor-settings";

// 加载
function loadEditorSettings(): EditorSettings {
  try {
    const saved = localStorage.getItem(EDITOR_SETTINGS_KEY);
    return saved ? { ...DEFAULT_EDITOR_SETTINGS, ...JSON.parse(saved) } : DEFAULT_EDITOR_SETTINGS;
  } catch {
    return DEFAULT_EDITOR_SETTINGS;
  }
}

// 保存
useEffect(() => {
  localStorage.setItem(EDITOR_SETTINGS_KEY, JSON.stringify(editorSettings));
}, [editorSettings]);
```

## 四、实现步骤

### 步骤 1：在 Settings.tsx 中定义编辑器设置

1. 定义 `EditorSettings` 接口和 `DEFAULT_EDITOR_SETTINGS`
2. 定义 `EDITOR_SETTINGS_KEY = "zmd-editor-settings"`
3. 导出 `EditorSettings` 类型和 `loadEditorSettings` 函数，供 App.tsx 和 VditorEditor.tsx 使用
4. 在 `SettingsTab` 类型中添加 `"editor"`：
   ```typescript
   type SettingsTab = "general" | "theme" | "shortcuts" | "editor" | "mindmap" | "image" | "about";
   ```
5. 在 `tabs` 数组中添加：
   ```typescript
   { id: "editor", label: "编辑器" },
   ```
6. 在主组件中添加 `editorSettings` 状态和保存逻辑
7. 实现 `EditorSettingsContent` 组件
8. 在 `settings-main` 中渲染：
   ```tsx
   {activeTab === "editor" && (
     <EditorSettingsContent settings={editorSettings} onChange={setEditorSettings} />
   )}
   ```

### 步骤 2：App.tsx 读取编辑器设置并传入 VditorEditor

1. 导入 `EditorSettings` 类型和 `loadEditorSettings`
2. 初始化 `viewMode` 和 `typewriterMode` 从编辑器设置中读取：
   ```typescript
   const [editorSettings, setEditorSettings] = useState(loadEditorSettings);

   const [viewMode, setViewMode] = useState<EditorMode>(editorSettings.defaultMode);
   const [typewriterMode, setTypewriterMode] = useState(editorSettings.typewriterMode);
   ```
3. 监听 localStorage 变化，设置面板保存后实时生效：
   ```typescript
   useEffect(() => {
     const handleStorage = (e: StorageEvent) => {
       if (e.key === "zmd-editor-settings") {
         const newSettings = loadEditorSettings();
         setEditorSettings(newSettings);
         setViewMode(newSettings.defaultMode);
         setTypewriterMode(newSettings.typewriterMode);
       }
     };
     window.addEventListener("storage", handleStorage);
     return () => window.removeEventListener("storage", handleStorage);
   }, []);
   ```
4. 将 `editorSettings` 传入 `VditorEditor`：
   ```tsx
   <VditorEditor
     ref={editorHandleRef}
     value={content}
     onChange={handleChange}
     mode={viewMode}
     theme={theme}
     typewriterMode={typewriterMode}
     editorSettings={editorSettings}  // ← 新增
     imageSettings={imageSettings}
     currentFilePath={fileName}
     activeVaultPath={...}
   />
   ```

### 步骤 3：VditorEditor.tsx 接收并应用编辑器设置

1. 在 `VditorEditorProps` 中添加 `editorSettings` 属性：
   ```typescript
   interface VditorEditorProps {
     // ...现有属性
     editorSettings?: EditorSettings;
   }
   ```

2. 在 Vditor 初始化时使用 `editorSettings` 替代硬编码值：
   ```typescript
   const es = editorSettings || DEFAULT_EDITOR_SETTINGS;

   const vditor = new Vditor(el, {
     mode,
     value: initialValue,
     cdn: "/vditor",
     icon: "ant",
     lang: "zh_CN",
     placeholder: "开始输入 Markdown... ✍️",
     theme: theme === "white" || theme === "mint" || theme === "liquid-glass" ? "classic" : "dark",
     height: "100%",
     width: "100%",
     typewriterMode,
     outline: { enable: es.outline, position: es.outlinePosition },
     counter: { enable: es.counter, type: es.counterType, max: es.counterMax || undefined, after: (length, counter) => { /* 更新底部栏字数 */ } },
     resize: { enable: es.resize },
     cache: { enable: es.cache },
     // ...其余配置不变

     preview: {
       mode: "editor",
       maxWidth: es.previewMaxWidth,
       theme: { current: ..., path: "/vditor/dist/css/content-theme" },
       hljs: {
         style: es.codeTheme === "auto"
           ? (theme === "white" || theme === "mint" || theme === "liquid-glass" ? "atom-one-light" : "atom-one-dark")
           : es.codeTheme,
         enable: true,
         lineNumber: es.codeLineNumber,
       },
       markdown: {
         autoSpace: es.autoSpace,
         gfmAutoLink: es.gfmAutoLink,
         fixTermTypo: es.fixTermTypo,
         footnotes: es.footnotes,
         toc: es.toc,
         paragraphBeginningSpace: es.paragraphBeginningSpace,
         sanitize: es.sanitize,
         codeBlockPreview: es.codeBlockPreview,
         mathBlockPreview: es.mathBlockPreview,
         mark: es.mark,
         sup: es.sup,
         sub: es.sub,
       },
       math: {
         engine: es.mathEngine,
       },
     },

     link: {
       isOpen: es.linkOpenNewTab,
       click: (href) => { /* 现有逻辑 */ },
     },
   });
   ```

3. **编辑器设置变化时的动态更新**：
   - `typewriterMode`、`outline`、`counter`、`resize`、`cache` 等部分选项需要重建 Vditor 实例才能生效 → 将 `editorSettings` 加入 Vditor 初始化的 `useEffect` 依赖数组（或使用 JSON 序列化做浅比较，避免频繁重建）
   - `preview.markdown.*`、`preview.hljs.*`、`preview.maxWidth` 等预览相关选项也需要重建
   - 实际实现中，可以只依赖 `editorSettings` 的 JSON 字符串变化来触发重建：

   ```typescript
   const editorSettingsKey = JSON.stringify(editorSettings);

   useEffect(() => {
     // ... Vditor 初始化逻辑
   }, [mode, editorSettingsKey]); // editorSettings 变化时重建
   ```

   > ⚠️ 重建会丢失撤销/重做栈和光标位置。对于需要重建的选项，在 UI 上添加提示「修改后需重新打开文件生效」，或在重建后自动恢复光标位置。

### 步骤 4：迁移通用设置中的编辑器相关项（可选）

将 `GeneralSettings` 中的 `fontSize`、`editorFont` 迁移到编辑器设置中，使「通用」标签专注于应用级设置，「编辑器」标签专注于编辑器行为。此步骤可选，不影响功能。

## 六、注意事项

### 6.1 重建策略

Vditor 的配置选项大多**只在初始化时生效**，运行时通过 `setTheme()` 等方法可动态修改的仅有：
- `theme` / `contentTheme` / `codeTheme`（通过 `setTheme()`）
- `typewriterMode`（通过 `vditor.options.typewriterMode = value` 直接赋值，但不一定实时生效）

因此大部分编辑器设置变更需要**重建 Vditor 实例**。重建逻辑已存在于 `VditorEditor.tsx`（`mode` 变化时重建），只需扩展依赖项即可。

**重建影响**：
- 撤销/重做栈被清空
- 光标位置丢失
- 编辑器闪烁（加载状态）

**缓解方案**：
1. 重建前保存当前内容和光标位置，重建后恢复
2. 对不需要重建的选项（如 `typewriterMode`）尝试动态修改
3. 在设置面板中对需要重建的选项标注「需重新打开文件生效」

### 6.2 与现有功能的兼容

- **编辑模式切换**：App.tsx 底部栏的 `cycleMode` 按钮和命令面板中的模式切换功能保持不变，`viewMode` 仍由 App.tsx 管理，`defaultMode` 仅作为初始值
- **打字机模式**：底部栏的打字机模式按钮保持不变，`typewriterMode` 仍由 App.tsx 管理，设置中的值作为初始值并双向同步
- **字数统计**：如果启用 `counter`，Vditor 会在编辑器底部显示字数，可能与 App.tsx 底部栏的字数显示重复。建议：
  - 启用 Vditor counter 时，隐藏 App.tsx 底部栏的字数显示
  - 或将 Vditor counter 的回调接入 App.tsx 底部栏

### 6.3 代码主题与应用主题的联动

当 `codeTheme === "auto"` 时：
- 浅色应用主题（white/mint/liquid-glass）→ `atom-one-light`
- 深色应用主题（catppuccin-mocha/mint-dark）→ `atom-one-dark`

当 `codeTheme` 为具体值时，使用用户指定的代码主题，不随应用主题变化。

### 6.4 安全性

`sanitize` 选项默认开启（XSS 过滤）。关闭后，Markdown 中的原始 HTML 将不被过滤。在设置面板中应添加警告提示：
> ⚠️ 关闭 XSS 过滤可能导致安全风险，仅在你信任所有 Markdown 来源时关闭。
